# 노드 예외 처리

## 노드 예외 처리 Exception

 예외란 보통 처리하지 못한 에러를 가리킵니다. 이런 에러들은 실행 중인 노드 프로세스를 멈추게 만듭니다. 노드는 싱글 스레드로 동작하기 때문에 이러한 에러를 처리하는 것이 중요합니다. 

따라서, **노드를 이용해 서버를 만들 때 에러 로그는 기록되지만 작업은 계속 진행될 수 있도록  해야합니다**. 에러는 다음과 같이 에러가 발생할 것 같은 부분에 **try-catch문**을 이용해 방지해줍니다. 

### try-catch를 이용한 예외 처리 

```javascript
try{
    에러가 발생할 것 같은 코드
}
catch(error){
    console.error(error)
}
```

 try-catch를 통해 에러를 다룰 경우 프로세스가 멈추지 않고 해당 에러를 로깅하며 프로세스를 유지할 수 있다. 

### process.on\(uncaughtException, \(errror\) =&gt; {}\)를 이용한 예외 처리

```javascript
process.on("uncaughtException", (error) => {
    console.error("예기치 못한 에러", error);
});
```

 process 객체에 uncaughtException 이벤트 리스너를 통해 처리하지 못한 에러가 발생했을 때 이벤트 리스너가 실행되고 프로세스가 유지되도록 합니다. 

 정말 예측이 불가능한 에러를 처리하기 위해 사용하고 되도록 try-catch에서 에러를 최대한 처리해주어야 합니다. 

{% hint style="info" %}
**서버 운영은 에러와의 싸움입니다**. 모든 에러 상황에 대비하는 것이 최선이지만, 시간이나 비용, 인력 등의 제약으로 대비하지 못한 에러가 있을 수 있습니다.   
따라서 **에러 발생 시 철저히 기록\(로깅\)하는 습관을 들이고, 주기적으로 로그를 확인하면서 보완해 나가야 합니다.** 
{% endhint %}
